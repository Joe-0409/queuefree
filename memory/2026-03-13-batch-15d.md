### 3. `services/api/prisma/seed.ts`
Seeds minimal demo data for repeatable testing:
- Cleans up `smoke_` prefixed idempotency records
- Deletes payments for demo orders
- Upserts `order_wait_pay_01` (WAIT_PAY status)
- Upserts `order_paid_01` (PAID status)
- Resets `user_demo_01` queue guard to VALID

### 4. `services/api/scripts/smoke-write-ops.ts`
Full smoke test suite with assert-based validation:
- Create order → 201
- Idempotency replay → 201 (cached)
- Idempotency mismatch → 400
- Payment intent missing header → 400
- Payment intent success → 201
- Payment intent replay → 201 (cached)
- Payment on PAID order → 409
- Queue guard missing header → 400
- Queue guard success → 200
- Queue guard replay → 200 (cached)
